Suggestions (theorycraft, not decided/started):

Advanced missile intelligence
  - Not yet done: a smart/evasive missile as its own data variant (e.g. ICBM_MISSILE_MK2 profile)
    - reuse MissileEntity's phases, just alter cruise altitude/add jink instead of a whole new
    flight system. Terrain-hugging and SAM evasion (below) both shipped as behavior on the one
    existing missile type instead, not a separate item.
  - Not yet done: a simpler alternative to full evasion AI - a radarCrossSection multiplier per
    missile type that shrinks the effective detection radius against it in radar/SAM/CIWS scans.
    Would pair naturally with the line-of-sight/altitude radar overhaul noted above.

Defensive missile speed / self-destruct timer
  - Add samMaxFlightTicks (or similar) so interceptors self-discard via the existing
    remove(RemovalReason) path, releasing CLAIMED_TARGETS automatically.
  - Decide if this is about balance (cap interceptor speed advantage) or realism (fuel/burn
    time) - if fuel, mirror MissileEntity's boost/cruise/terminal phase structure on interceptors
    for symmetry.

Radar tiers
  - Differentiate tiers beyond radius: scan interval (currently a shared SCAN_INTERVAL_TICKS,
    would need to go per-tier), position fidelity (lower tiers jitter/round contact coords,
    higher tiers exact), and outgoing-track range (only top tier keeps whole-map outgoing
    tracking, lower tiers lose the track past their own radius).
  - Open question: if multiple radars ever coexist, do SAM/CIWS care which radar they're wired
    to (tier/quality) or just that WireNetwork finds *a* radar at all (current behavior)?

Satellite (map-wide detection)
  - Simplest route, most consistent with existing patterns: a placeable "satellite uplink" block,
    basically a radar tier with unlimited radius and no wire requirement - reuses
    RadarBlockEntity's scan/push logic almost as-is.
  - More thematic but bigger lift: an actually-launched entity that settles into "orbit" and
    feeds a linked ground station - needs its own entity type, an ownership/link model, and a
    decision on whether it's shootable (SAM/CIWS only scan MissileEntity's registry today, so a
    satellite would need to opt in or stay immune).

Missile Silo - display loaded missile
  - Rendering-only, no new gameplay logic. Reuse the ItemModelManager.updateForNonLivingEntity
    call already solved in MissileEntityRenderer, just with fixed orientation instead of
    velocity-derived yaw/pitch. Good low-risk first pick since it shares code with something
    already built.

Physical EMPs
  - Thrown/placed charging entity, similar shape to a missile/interceptor rather than a block.
  - Cheapest way to "disable defenses": a per-block-entity empDisabledUntilTick gate checked
    alongside SAM/CIWS's existing ammo/wire/cooldown checks in tick() - additive and reversible,
    no need to actually sever/destroy the WIRE network.
  - Charging noise: periodic world.playSound on a tick counter - would be the mod's first
    looping-while-active sound, nothing else does this yet.

Other ideas not yet in the roadmap
  - Radar jamming - EMP/jammer feeds radar false contacts or blanks the scope; cheap since
    RadarUpdatePayload is just a data push to spoof.
  - Salvo/MIRV fire - reuse a launcher's existing per-block waypoint list as a target queue for
    multiple missiles instead of one target per launch.
  - Ammo-empty alert for SAM/CIWS - currently a silent no-op per CLAUDE.md; a simple particle/
    visual "out of ammo" state would close that gap cheaply.
